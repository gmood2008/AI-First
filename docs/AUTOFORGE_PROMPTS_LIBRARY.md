# AutoForge Prompts Library

## 📚 推荐描述模板

本文档提供了针对不同场景的推荐需求描述模板，帮助您更准确地生成能力。

---

## 🌐 网络请求类

### API 数据获取

**模板：**
```
从 {API名称} 获取 {数据类型}，需要 {认证方式}
```

**示例：**
- "从 CoinGecko API 获取比特币价格，需要 API key"
- "从 GitHub API 获取仓库信息，需要 OAuth token"
- "从 Slack API 获取频道列表，需要 Bot token"

**完整示例：**
```bash
forge create "从 CoinGecko API 获取比特币价格，需要 API key" \
  --reference ./docs/coingecko_api.md \
  --id "net.crypto.get_price"
```

### API 数据发送

**模板：**
```
向 {API名称} 发送 {数据类型}，使用 {HTTP方法}
```

**示例：**
- "向 Slack API 发送消息到指定频道，使用 POST 方法"
- "向 GitHub API 创建 Issue，使用 POST 方法"
- "向 Jira API 更新任务状态，使用 PUT 方法"

## 💡 最佳实践

### 1. 明确指定细节

**❌ 不好的描述：**
```
"处理数据"
```

**✅ 好的描述：**
```
"从 CSV 文件读取数据，转换为 JSON 格式，并验证数据完整性"
```

### 2. 包含认证信息

**❌ 不好的描述：**
```
"获取 GitHub 仓库信息"
```

**✅ 好的描述：**
```
"使用 OAuth token 从 GitHub API 获取仓库信息，包括名称、描述和星标数"
```

### 3. 指定错误处理

**❌ 不好的描述：**
```
"发送邮件"
```

**✅ 好的描述：**
```
"使用 SMTP 发送邮件，包含重试机制和错误处理"
```

### 4. 使用参考文档

**推荐：**
```bash
forge create "调用公司内部 API" \
  --reference ./docs/internal_api.md \
  --reference ./examples/auth_example.py
```

## 🎯 场景模板

### 场景 1: 快速 API 集成

```bash
forge create "从 {API} 获取 {数据}，需要 {认证}" \
  --reference ./docs/api_docs.md \
  --id "net.{service}.get_{data}"
```

### 场景 2: 数据处理管道

```bash
forge create "读取 {源}，转换为 {格式}，写入 {目标}" \
  --test-first \
  --id "data.{source}_to_{target}"
```

### 场景 3: 监控和告警

```bash
forge create "监控 {指标}，当 {条件} 时 {操作}" \
  --reference ./docs/alert_config.md \
  --id "monitor.{metric}_alert"
```
